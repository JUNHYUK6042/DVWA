# Command Injection

- 본 실습은 DVWA(Damn Vulnerable Web Application) 환경에서 Command Injection 공격을 수행하고,  
이를 주요정보통신기반시설 기술적 취약점 분석·평가 기준에 따라 분석하는 것을 목적으로 합니다.

---

## 주요정보통신기반시설 기준

> 출처 : https://www.kisa.or.kr/2060204/form?postSeq=22&page=1#fnPostAttachDownload ( p.679 ~ 692)

### 점검 내용

- 시스템 명령어 실행 시 사용자 입력값 검증 여부 점검

---

### 점검 목적

- 외부 입력값을 통한 비인가 시스템 명령어 실행 방지
- 서버 내부 정보 유출 및 시스템 제어 권한 탈취 방지

---

### 보안 위협

- 사용자 입력값을 검증하지 않을 경우 시스템 명령어 실행 가능
- 서버 내부 파일 정보(/etc/passwd 등) 유출 가능
- 공격자가 서버 권한으로 임의 명령 실행 가능
- 시스템 장악 및 추가 공격으로 확장 가능

---

### 판단 기준

- 양호: 사용자 입력값 검증 및 명령어 실행 제한 적용
- 취약: 입력값 검증 없이 시스템 명령어 실행 가능

---

## 점검 절차

1. 입력값을 받는 기능 존재 여부 확인  
2. 정상 입력값 동작 확인 (ping 등)  
3. 명령어 구분자를 이용한 추가 명령 실행 시도  
4. 시스템 명령어 실행 여부 확인  
5. 결과 출력 및 정보 노출 여부 확인  

---

### 정상 동작 확인

- IP 입력창에 `127.0.0.1` 입력 후 Submit 버튼 클릭 후 명령어를 실행합니다.
```
127.0.0.1
```

![01](/Command%20Injection/img/01.png)

- 정상적으로 ping 명령이 실행되며 결과가 출력되는 것을 확인했습니다.

---

### 시스템 계정정보 파일 조회

- 입력값에 다음 명령어를 입력합니다.
```
; cat /etc/passwd
```

![02](/Command%20Injection/img/02.png)

- ping 명령과 함께 `/etc/passwd` 파일 내용이 출력되는 것을 확인했습니다.
  - 사용자 계정 목록
  - 로그인 가능 여부
  - 기본 쉘 정보
- 이는 사용자 입력값이 필터링 없이 시스템 명령어에 그대로 전달된 것을 의미합니다.

---

### 사용자 조회

- 입력값에 다음 명령어를 입력합니다.
```
; whoami
```

![03](/Command%20Injection/img/03.png)

- 결과로 `www-data`가 출력되었으며,  
  웹 서버 권한으로 명령어가 실행된 것을 확인했습니다.

---

### 디렉토리 조회

- 입력값에 다음 명령어를 입력합니다.
```
& ls -al
```

![04](/Command%20Injection/img/04.png)

- 서버 디렉토리 내부 파일 목록이 출력되는 것을 확인했습니다.

---

### 취약점 분석

- 실습을 진행하면서 확인해보니, 해당 시스템은 사용자 입력값에 대한 검증이 전혀 이루어지지 않았습니다.

- 입력값에 `;`와 같은 명령어 구분자를 삽입했을 때,  
  기존 명령어와 함께 추가 명령이 그대로 실행되는 것을 확인했습니다.

- `/etc/passwd` 파일 조회, `whoami`, `ls` 등의 명령이 정상적으로 실행되는 것으로 보아  
  서버 내부 정보 접근 및 시스템 제어가 가능한 상태였습니다.

- 특히 `whoami` 결과를 통해 웹 서버 권한(`www-data`)으로 명령이 실행되는 것을 확인할 수 있었습니다.

- 이러한 점을 종합해보면, 사용자 입력값 검증이 이루어지지 않아  
  시스템 명령어 실행이 가능한 취약한 상태로 판단됩니다.

- 실제 환경이었다면 서버 내부 정보 유출 및 추가적인 시스템 침해로 이어질 가능성이 높습니다.

- 따라서 본 시스템은 입력값 검증이 미흡한 상태로, 보안 강화가 필요한 것으로 판단됩니다.

---

## 대응 방안

### 입력값 검증 (Input Validation)

- 사용자 입력값에 대해 허용된 값만 입력받도록 제한합니다.
- IP 입력 시 숫자(0-9)와 점(.)만 허용하도록 필터링합니다.

#### 핵심 효과
- 악의적인 명령어 삽입을 차단합니다.

---

### 특수문자 필터링

- `;`, `&`, `|`, `&&`, `||` 등의 명령어 구분자를 차단합니다.
- 위험 문자열 필터링 적용

#### 핵심 효과
- 추가 명령 실행을 방지합니다.

---

### 안전한 명령어 실행 방식 적용

- 시스템 명령어 실행 시 사용자 입력값을 직접 연결하지 않도록 합니다.
- shell 실행 함수 사용을 지양하고 안전한 API를 사용합니다.
- 입력값을 인자로 전달하는 방식으로 처리합니다.

#### 핵심 효과
- Command Injection 원천 차단합니다.

---

### 권한 최소화 (Least Privilege) 

- 웹 서버 실행 계정 권한을 최소화합니다.
- 중요 파일 접근을 제한 합니다.

#### 핵심 효과 
- 침해 발생 시 피해를 최소화합니다.

---

### 웹 방화벽 적용

- 명령어 패턴 탐지 및 차단합니다.
- 특수문자 필터링 룰을 적용합니다.

#### 핵심 효과
- 공격 시도 사전에 차단합니다.
