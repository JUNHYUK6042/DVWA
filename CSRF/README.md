# CSRF

## 개요
CSRF(Cross-Site Request Forgery)는 사용자가 의도하지 않은 요청을 서버에 보내도록 유도하는 공격 방식입니다.

---

## 핵심 개념
- 사용자가 로그인된 상태를 악용하여 공격이 수행됩니다.
- 서버는 요청이 정상 사용자로부터 온 것인지 구별하지 못합니다.
- 사용자의 권한으로 공격자가 원하는 요청이 실행됩니다.

---

## 공격 방식 설명
공격자는 악성 페이지를 만들어 사용자가 해당 페이지에 접속하도록 유도합니다.  
사용자가 로그인된 상태라면, 브라우저가 자동으로 인증 정보(쿠키 등)를 포함하여 요청을 전송하게 됩니다.  
이로 인해 사용자의 의도와 상관없이 특정 기능(비밀번호 변경, 게시글 작성 등)이 수행됩니다.

---

## 예시
- 비밀번호 변경 공격 예시  
  → 사용자가 로그인된 상태에서 공격자가 만든 페이지 접속  
  → `<img src="http://target.com/change_pw?pw=1234">`  
  → 사용자 의도와 관계없이 비밀번호 변경 요청 실행

- 송금 요청 예시  
  → `<form action="http://bank.com/transfer" method="POST">`  
  → 자동 제출을 통해 금액 이체 수행

---

## 특징
- 사용자의 인증 상태를 그대로 이용합니다.
- 별도의 취약점 없이도 발생할 수 있습니다.
- GET/POST 요청 모두 공격이 가능합니다.

---

## 대응 방안 (간단)
- CSRF 토큰 적용
- Referer 또는 Origin 검증
- 중요 요청 시 추가 인증 요구

---

## 정리
- CSRF는 사용자의 인증 정보를 이용하여 의도하지 않은 요청을 수행하게 만드는 공격 방식입니다.
