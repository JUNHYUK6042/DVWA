# Blind SQL Injection

## 개요

- Blind SQL Injection은 SQL Injection의 한 종류로, 에러 메시지나 결과가  
  직접적으로 보이지 않는 상황에서도 데이터베이스의 정보를 추론할 수 있는 공격 방식입니다.

- 일반적인 SQL Injection과 다르게 결과가 바로 출력되지 않기 때문에  
  참/거짓(True/False) 또는 시간 지연을 이용해 하나씩 정보를 확인하는 방식으로 진행됩니다.

---

## Blind SQL Injection 유형

| 구분 | 설명 | 특징 |
|------|------|------|
| Boolean Based | 참/거짓 조건으로 판단 | 응답 내용 차이로 정보 확인 |
| Time Based | 시간 지연으로 판단 | sleep() 등으로 결과 확인 |

---

## 핵심 개념

- 결과가 직접 출력되지 않는 환경에서도 공격이 가능합니다.  
- 조건문을 이용해 참인지 거짓인지 확인하면서 데이터를 하나씩 추출합니다.  
- 일반 SQL Injection보다 느리지만, 필터링이 있어도 우회가 가능한 경우가 많습니다.  

---

## 공격 방식 설명

- 공격자는 조건문을 이용해 특정 값이 맞는지 확인합니다.  
- 예를 들어 관리자 비밀번호의 첫 글자가 'a'인지 확인하는 식으로 접근합니다.

- 결과가 참이면 정상 응답, 거짓이면 다른 응답이 나오기 때문에  
  이를 비교하면서 데이터를 하나씩 추출합니다.

- 또는 sleep() 함수를 이용해 응답 시간을 지연시켜  
  조건이 맞는지 판단하는 방식도 사용합니다.

---

## 예시

- Boolean Based 공격
```
- `' AND 1=1 --`  (참 → 정상 응답)  
- `' AND 1=2 --`  (거짓 → 다른 응답)  

- `' AND SUBSTRING(password,1,1)='a' --` (비밀번호 첫 글자가 a인지 확인)
```

- Time Based 공격
```
- `' AND SLEEP(5) --` (조건이 참일 경우 5초 지연)

- `' AND IF(SUBSTRING(password,1,1)='a', SLEEP(5), 0) --`  
```

---

## 특징

- 에러 메시지가 없어도 공격이 가능합니다.  
- 응답 내용이나 시간 차이를 이용해 데이터를 추출합니다.  
- 자동화 도구(sqlmap 등)를 사용할 경우 빠르게 정보 탈취가 가능합니다.  
- 탐지하기 어렵고, 장시간에 걸쳐 공격이 진행될 수 있습니다.  

---

## 대응 방안

- Prepared Statement를 사용하여 입력값과 SQL을 분리합니다.  
- 사용자 입력값에 대한 검증을 수행합니다.  
- 동일한 요청에 대해 항상 동일한 응답을 반환하도록 설정합니다.  
- 응답 시간 차이가 발생하지 않도록 처리합니다.  
- 데이터베이스 에러 메시지를 외부에 노출하지 않도록 설정합니다.  
- WAF를 통해 반복적인 요청 및 패턴을 탐지합니다.  

---

## 정리

- Blind SQL Injection은 결과가 보이지 않는 환경에서도  
  조건 비교를 통해 데이터를 추출하는 공격 방식입니다.  

- 응답 차이 또는 시간 지연을 이용하기 때문에  
  탐지하기 어렵고 실제 서비스에서 더 위험하게 사용될 수 있습니다.
