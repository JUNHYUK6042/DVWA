# SQL Injection

## 개요

- SQL Injection은 웹 애플리케이션이 사용자 입력값을 검증하지 않고 SQL 쿼리에 그대로 포함할 때 발생하는 취약점입니다.  
- 공격자는 이를 이용하여 데이터베이스에 비정상적인 쿼리를 실행하고,  
  로그인 우회, 정보 조회, 데이터 수정 및 삭제 등의 공격을 수행할 수 있습니다.  

---

## SQL Injection 취약점 유형

| 구분 | 설명 | 특징 |
|------|------|------|
| Login Bypass | 인증 우회 공격 | 관리자 페이지 접근 가능 |
| Union Based | UNION 구문 이용 | 다른 테이블 정보 조회 |
| Error Based | 에러 메시지 활용 | DB 정보 노출 |
| Blind SQLi | 참/거짓 기반 추론 | 응답 차이로 정보 확인 |
| Time Based | 시간 지연 기반 추론 | sleep() 등으로 판단 |

---

## 핵심 개념

- 사용자 입력값이 SQL 쿼리에 직접 들어갈 때 발생합니다.  
- 입력값 검증이 없거나 Prepared Statement를 사용하지 않을 경우 공격이 가능합니다.  
- 단순 로그인 페이지뿐 아니라 검색, 게시판, 회원조회 등 다양한 곳에서 발생할 수 있습니다.  

---

## 공격 방식 설명

- 공격자는 입력값에 `'`, `--`, `or 1=1`, `union select` 등의 SQL 구문을 삽입합니다.  
- 이를 통해 원래 의도하지 않은 SQL이 실행되도록 만들어 인증을 우회하거나 데이터를 탈취합니다.  

---

## 예시

- 정상 입력  
  → admin  
  → user01  

- 공격 입력  
  → `' OR 1=1 --`  
  → `admin' --`  
  → `' UNION SELECT 1,2,3 --`  
  → `' AND SLEEP(5) --`  

- 웹 입력 예시  
  → 로그인 ID 입력창에  
  `admin' OR '1'='1` 입력 후 로그인 우회  

---

## 특징

- 데이터베이스 내부 정보를 직접 조회할 수 있습니다.  
- 관리자 계정 탈취 및 인증 우회가 가능합니다.  
- 회원정보, 비밀번호, 개인정보 유출로 이어질 수 있습니다.  
- 데이터 수정 및 삭제가 가능하여 서비스 장애가 발생할 수 있습니다.  
- 다른 공격 권한 상승으로 이어질 가능성이 높습니다.  

---

## 대응 방안

- 사용자 입력값에 대한 검증을 수행합니다.  
- `'`, `"`, `;`, `--`, `#` 등의 특수문자를 필터링합니다.  
- Prepared Statement(Parameterized Query)를 사용합니다.  
- 데이터베이스 에러 메시지가 사용자에게 노출되지 않도록 설정합니다.  
- DB 계정에 최소 권한만 부여합니다.  
- WAF를 적용하여 SQL Injection 패턴을 탐지 및 차단합니다.  

---

## 정리

- SQL Injection은 입력값 조작을 통해 데이터베이스를 공격하는 대표적인 웹 취약점입니다.  
- 인증 우회부터 정보 유출, 데이터 삭제까지 가능하기 때문에 위험도가 매우 높습니다.  
- 입력값 검증과 Prepared Statement 적용이 가장 중요한 대응 방법입니다.
