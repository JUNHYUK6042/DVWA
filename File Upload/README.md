# File Upload

## 개요

- File Upload 취약점은 웹 애플리케이션이 사용자로부터 업로드된 파일을 검증 없이 저장하거나 실행 가능한 위치에 업로드할 때 발생하는 취약점입니다.  
- 공격자는 이를 이용하여 악성 파일을 업로드하고, 서버에서 실행시켜 시스템 권한을 획득하거나 추가 공격을 수행할 수 있습니다.  

---

## File Upload 취약점 유형

| 구분 | 설명 | 특징 |
|------|------|------|
| 웹쉘 업로드 | 악성 스크립트 파일 업로드 | 서버 명령어 실행 가능 |
| 확장자 우회 | 필터링 우회 후 파일 업로드 | .php → .php.jpg 등 |
| MIME 타입 우회 | Content-Type 속여 업로드 | 서버 검증 미흡 시 가능 |
| 실행 위치 문제 | 업로드 경로에서 실행 가능 | 업로드 즉시 RCE 가능 |
| 파일 덮어쓰기 | 기존 파일 덮어쓰기 | 서비스 장애 및 변조 |

---

## 핵심 개념

- 파일 업로드 자체는 정상 기능이지만, 검증이 부족할 경우 공격 벡터로 사용됩니다.  
- 업로드된 파일이 서버에서 실행 가능한 위치에 저장되면 매우 위험합니다.  
- 단순 확장자 체크만으로는 충분한 보안이 되지 않습니다.  

---

## 공격 방식 설명

- 공격자는 업로드 기능을 통해 웹쉘(Web Shell) 또는 악성 파일을 서버에 업로드합니다.  
- 이후 업로드된 파일에 접근하여 시스템 명령을 실행하거나 내부 정보를 탈취합니다.  

---

## 예시

- 정상 업로드  
  → image.jpg  
  → profile.png  

- 공격 업로드  
  → shell.php  
  → shell.php.jpg  
  → shell.phtml  
  → shell.php%00.jpg  

- 요청 예시  
  → 파일 업로드 후 `http://target.com/uploads/shell.php` 접근  

---

## 특징

- 업로드된 파일을 통해 서버에 직접 접근이 가능합니다.  
- 웹쉘 업로드 시 서버 권한으로 명령어 실행이 가능합니다.  
- 설정 파일, 소스 코드 등 내부 정보 탈취로 이어질 수 있습니다.  
- 지속적인 접근(Backdoor) 확보가 가능하여 장기적인 침해로 이어질 수 있습니다.  
- 다른 취약점(LFI, RCE 등)과 연계될 가능성이 높습니다.  

---

## 대응 방안

- 사용자 업로드 파일에 대한 확장자 검증을 수행합니다.  
- 파일 내용(Content) 검증을 통해 실제 파일 형식을 확인합니다.  
- 업로드 파일을 실행 불가능한 경로에 저장합니다.  
- 업로드 파일명은 랜덤값으로 변경하여 저장합니다.  
- 업로드 디렉토리에서 스크립트 실행을 차단합니다.  
- 최소 권한 원칙을 적용하여 서버 접근 범위를 제한합니다.  

---

## 정리

- File Upload 취약점은 악성 파일 업로드를 통해 서버 권한을 획득할 수 있는 매우 위험한 취약점입니다.  
- 단순 확장자 검증만으로는 방어가 불가능하며,  
  저장 위치, 실행 여부, 파일 내용 검증까지 함께 고려해야 합니다.
